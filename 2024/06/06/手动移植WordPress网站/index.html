手动移植WordPress网站

2024-06-06

  1. 网站文件：

    • WordPress 核心文件：包括所有的 WordPress 核心文件，如 wp-admin、wp-includes 文件夹及其内容。
    • 主题文件：位于 wp-content/themes 目录下，包含你当前使用的主题及任何自定义主题文件。
    • 插件文件：位于 wp-content/plugins 目录下，包含所有安装和激活的插件。
    • 上传文件：位于 wp-content/uploads 目录下，包含所有的媒体文件（如图片、视频等）。

  2. 数据库：

    • 数据库导出文件：使用 phpMyAdmin 或类似工具，将整个 WordPress 数据库导出为 SQL 文件。这个数据库包含了所有的文章、页面、评论、用户信息、插件设置等。

  3. 配置文件：

    • wp-config.php 文件：该文件位于 WordPress 根目录下，包含了数据库连接信息和其他重要配置。
    • .htaccess 文件：如果你有自定义的 .htaccess 文件（通常用于 URL 重写规则等），也需要拷贝。

移植步骤概述：

  1. 备份源网站：确保备份网站的所有文件和数据库。
  2. 将文件上传到新服务器：将拷贝的文件上传到新服务器上的相应位置。
  3. 导入数据库：在新服务器上创建一个新的数据库，并导入之前导出的数据库文件。
  4. 修改配置：
    • 更新 wp-config.php 文件中的数据库连接信息以匹配新服务器的数据库配置。
    • 根据需要修改 .htaccess 文件。
  5. 更新 URL：
    • 如果域名发生变化，需要在数据库中更新 URL，可以使用 phpMyAdmin 或类似工具进行批量替换，或者使用插件如 Search and Replace 进行处理。
  6. 测试网站：确保所有功能正常运行，包括主题、插件和链接等。
